# Coordit Roadmap

문서 상태: 모바일 MVP 기준 정리본  
기준일: 2026-06-25

## 1. 문서 목적

이 문서는 Coordit의 개발 계획과 단계별 우선순위를 정리한다.

이 문서는 API 상세, DB schema, Fit Engine 계산식을 설명하지 않는다.

## 2. 현재 상태

현재 Coordit은 기준 의류 기반 핏 추천 MVP의 기본 골격을 갖추고 있다.

구현된 핵심 영역:

- 회원가입/로그인
- 사용자 프로필 조회/수정
- 보유 의류 관리
- 보유 의류 실측값 관리
- 기준 의류 관리
- 외부 상품 관리
- 외부 상품 사이즈표 관리
- 단일/다중 기준 의류 기반 추천
- 추천 결과 저장
- 추천 로그 저장
- 사용자 피드백 저장
- Rule-based Fit Engine

부분 구현 또는 MVP 수준:

- URL 기반 상품 생성은 실제 크롤링이 아니라 mock 동작이다.
- 신체 치수는 저장되지만 핵심 추천 점수에는 적극 반영되지 않는다.
- 피드백은 저장되지만 추천 가중치 학습에는 아직 사용되지 않는다.
- OCR 파이프라인은 없다.
- 브랜드별 보정은 없다.
- 운영 수준의 테스트, 배포, 모니터링은 보강이 필요하다.

## 3. Phase 1. Mobile MVP

목표: 모바일 앱에서 첫 추천 플로우를 끝까지 완성한다.

범위:

- 4탭 앱 구조 정리
- Home 추천 준비 상태
- Closet 보유 의류 등록
- Closet 실측값 입력
- Closet 기준 의류 지정
- Fit Lab 외부 상품 등록
- Fit Lab 사이즈표 수동 입력
- Fit Lab 추천 실행
- 추천 결과 상세
- Styling 추천 이력
- 피드백 등록

API 보강 후보:

- Home 요약 API
- 추천 준비 상태 API
- 사이즈표 bulk 입력 API
- 카테고리 메타데이터 API

완료 기준:

- 신규 사용자가 가입 후 첫 추천 결과까지 도달할 수 있다.
- 추천 실행 전 부족한 데이터 상태를 앱에서 안내할 수 있다.
- 추천 결과의 사이즈, 점수, confidence, 부위별 차이를 확인할 수 있다.
- 추천 결과에 피드백을 남길 수 있다.

## 4. Phase 2. Beta 안정화

목표: 제한된 사용자에게 테스트 가능한 안정적인 Beta를 만든다.

범위:

- 모바일 UX 정리
- API 에러 응답 표준화
- 인증 만료 처리 강화
- 추천 결과 설명 개선
- 피드백 플로우 개선
- 통합 테스트 추가
- Supabase migration 운영 방식 정리
- 운영 환경변수 관리
- CORS 정책 제한
- 기본 로깅/모니터링

완료 기준:

- 주요 사용자 플로우에 회귀 테스트가 있다.
- 추천 실패 케이스가 앱에서 명확히 안내된다.
- 운영 환경 배포가 가능하다.

## 5. Phase 3. OCR

목표: 사용자가 사이즈표 이미지를 직접 입력하지 않아도 되도록 자동 추출을 도입한다.

범위:

- 사이즈표 이미지 업로드
- OCR 텍스트 추출
- 추출 결과 검수 화면
- 측정값 필드 매핑
- 추출 신뢰도 저장
- 수동 수정 플로우

고려 사항:

- OCR 결과는 바로 추천에 쓰기보다 사용자의 확인을 거친다.
- 추출 신뢰도는 추천 confidence에 반영할 수 있다.
- 쇼핑몰마다 사이즈표 형식이 다르므로 정규화 계층이 필요하다.

## 6. Phase 4. URL Parsing

목표: 상품 URL 입력만으로 상품 정보와 사이즈표 후보를 가져온다.

범위:

- 상품명 추출
- 브랜드/쇼핑몰명 추출
- 상품 이미지 추출
- 카테고리 후보 추론
- 사이즈표 텍스트 파싱
- 쇼핑몰별 파서 전략
- 실패 시 수동 입력 fallback

고려 사항:

- 현재 URL 기능은 mock이므로 실제 scraping/parsing과 명확히 구분한다.
- 법적/정책적 이슈가 있는 사이트는 파싱 대상에서 제외할 수 있다.
- 파싱 결과에는 출처와 신뢰도를 남긴다.

## 7. Phase 5. AI Recommendation

목표: Rule-based 추천을 사용자 피드백과 상품 데이터 기반 추천으로 확장한다.

범위:

- 피드백 기반 사용자별 선호 여유분 학습
- 브랜드별 사이즈 편차 보정
- 카테고리별 가중치 개선
- fit type penalty 개선
- 파싱 신뢰도 기반 confidence 개선
- 신체 치수와 기준 의류 데이터 결합

고려 사항:

- 충분한 피드백 데이터가 쌓이기 전까지는 rule-based 엔진을 유지한다.
- ML 또는 통계 모델은 추천 설명 가능성을 해치지 않아야 한다.
- 사용자가 이해할 수 있는 부위별 설명은 계속 제공해야 한다.

## 8. Phase 6. Styling 확장

목표: 핏 추천 이후의 사용 경험을 스타일링 영역으로 확장한다.

범위:

- 보유 의류 기반 룩 추천
- 저장한 룩 관리
- 추천 상품과 보유 의류 조합
- TPO 기반 추천
- 날씨 기반 추천

고려 사항:

- Styling은 초기 MVP의 핵심 기능이 아니다.
- 핏 추천 데이터와 자연스럽게 연결되는 기능부터 확장한다.

## 9. Phase 7. B2B 확장

목표: 쇼핑몰 또는 브랜드가 사용할 수 있는 사이즈 추천 인프라로 확장한다.

범위:

- 쇼핑몰 상품 상세 페이지용 추천 위젯
- 브랜드별 사이즈 데이터 분석
- 카테고리별 반품 위험 분석
- 구매 전환 및 반품률 개선 리포트
- API 또는 SDK 제공

고려 사항:

- B2B 확장은 사용자 데이터 보안과 동의 정책이 선행되어야 한다.
- 브랜드/쇼핑몰별 데이터 품질 차이를 흡수할 수 있어야 한다.

## 10. 장기 과제

- 데이터 품질 모니터링
- 추천 결과 A/B 테스트
- 개인화 모델 평가 지표
- privacy-first 데이터 정책
- 추천 설명 UX 고도화
- 다국가 사이즈 체계 대응
