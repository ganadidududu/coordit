# Coordit Product Overview

문서 상태: 모바일 MVP 기준 정리본  
기준일: 2026-06-25

## 1. 제품 소개

Coordit은 사용자가 이미 가지고 있고 잘 맞는 옷의 실측값을 기준으로, 온라인 상품의 사이즈표를 비교해 가장 가까운 사이즈를 추천하는 모바일 핏 추천 앱이다.

핵심 질문은 다음이다.

> 내가 가진 잘 맞는 옷과 비교했을 때, 지금 사려는 상품은 어떤 사이즈를 선택해야 할까?

초기 Coordit은 코디 추천 앱이 아니라 사이즈 실패를 줄이는 핏 의사결정 도구다. 날씨, TPO, 스타일링 큐레이션보다 기준 의류 실측값과 외부 상품 사이즈표 비교에 집중한다.

## 2. 문제 정의

온라인 의류 구매에서 사용자는 다음 문제를 반복해서 겪는다.

- 브랜드마다 같은 사이즈라도 실제 치수가 다르다.
- 쇼핑몰의 모델 착용 정보만으로 내 핏을 예측하기 어렵다.
- 사용자가 이미 잘 맞는 옷을 가지고 있어도 구매 판단에 체계적으로 활용하지 못한다.
- 사이즈표를 봐도 어떤 부위가 얼마나 차이 나는지 빠르게 판단하기 어렵다.
- 구매 후 실패 경험이 다음 추천에 반영되지 않는다.

Coordit은 사용자의 기준 의류 데이터를 중심으로 이 문제를 해결한다.

## 3. 핵심 가치

### 3.1 내 옷 기준 추천

사용자는 추상적인 S/M/L이 아니라 자신이 이미 입어보고 만족한 옷을 기준으로 새 상품을 비교한다.

### 3.2 실측 기반 판단

추천은 취향 문구나 모델 착용 컷이 아니라 실제 측정값 차이를 기반으로 한다.

### 3.3 부위별 설명

추천 사이즈만 보여주지 않고 어깨, 가슴, 총장, 허리, 엉덩이, 밑위, 바깥기장 등 부위별 차이를 함께 보여준다.

### 3.4 피드백 루프

사용자가 구매 후 남긴 실제 핏 피드백은 향후 추천 고도화의 기반 데이터가 된다.

## 4. 모바일 앱 구조

앱은 4개의 메인 탭으로 구성한다.

| 탭 | 역할 |
| --- | --- |
| `Home` | 추천 준비 상태, 최근 결과, 다음 액션을 보여주는 시작 화면 |
| `Closet` | 보유 의류, 실측값, 기준 의류를 관리하는 화면 |
| `Fit Lab` | 외부 상품과 사이즈표를 입력하고 핏 추천을 실행하는 화면 |
| `Styling` | 추천 결과 이력, 피드백, 저장한 룩을 관리하는 화면 |

상단에는 메뉴, Coordit 로고, 프로필 진입점을 둔다. 하단에는 4개 탭 네비게이션을 고정한다.

## 5. 핵심 사용자 플로우

### 5.1 첫 추천 플로우

1. 사용자가 회원가입 또는 로그인한다.
2. 사용자가 잘 맞는 보유 의류를 등록한다.
3. 사용자가 해당 의류의 실측값을 입력한다.
4. 사용자가 그 의류를 기준 의류로 지정한다.
5. 사용자가 구매하려는 외부 상품을 등록한다.
6. 사용자가 외부 상품의 사이즈표를 입력한다.
7. 사용자가 Fit Lab에서 추천을 실행한다.
8. 앱이 추천 사이즈, 점수, 신뢰도, 부위별 차이를 보여준다.
9. 사용자가 구매 또는 착용 후 피드백을 남긴다.

### 5.2 재사용 플로우

1. 사용자가 Home에서 추천 준비 상태와 최근 결과를 확인한다.
2. 사용자가 Fit Lab에서 새 상품을 추가한다.
3. 사용자가 이미 등록한 기준 의류를 선택한다.
4. 사용자가 사이즈표를 입력하고 추천을 실행한다.
5. 사용자가 Styling에서 추천 이력과 피드백을 관리한다.

## 6. 주요 기능

### 6.1 Auth

- 회원가입
- 로그인
- 로그아웃
- 인증 만료 처리

### 6.2 Profile

- 사용자 기본 프로필 조회
- 표시 이름, 성별, 출생연도 수정
- 신체 치수 등록 및 조회

신체 치수는 MVP에서 보조 데이터이며, 핵심 추천 계산은 기준 의류 실측값을 중심으로 한다.

### 6.3 Closet

- 보유 의류 등록, 수정, 삭제
- 보유 의류 실측값 입력, 수정, 삭제
- 카테고리별 의류 관리
- 기준 의류 지정
- 기준 의류 선호도 관리

### 6.4 Fit Lab

- 외부 상품 등록
- 상품 URL 저장
- 외부 상품 사이즈표 입력
- 기준 의류 선택
- 단일 또는 다중 기준 의류 기반 추천
- 추천 결과 상세 확인

Fit Lab은 Coordit의 핵심 화면이다.

### 6.5 Styling

- 최근 추천 결과 확인
- 추천 결과 상세 재확인
- 구매/착용 피드백 등록
- 저장한 스타일링 결과 확인

MVP에서는 Styling을 추천 이후의 이력과 피드백 중심 화면으로 사용한다. 코디 생성 기능은 확장 후보로 둔다.

## 7. MVP 범위

### 7.1 포함

- 이메일 기반 회원가입/로그인
- 사용자 프로필 관리
- 보유 의류 관리
- 보유 의류 실측값 관리
- 기준 의류 관리
- 외부 상품 관리
- 외부 상품 사이즈표 수동 입력
- Fit Lab 추천 실행
- 추천 결과 상세
- 최근 추천 결과 이력
- 추천 결과 피드백

### 7.2 제외

- 실제 쇼핑몰 URL 크롤링
- 사이즈표 이미지 OCR
- 자동 상품 정보 파싱
- ML 기반 추천
- 날씨 기반 코디 추천
- TPO 기반 코디 추천
- 커뮤니티, 팔로우, 공유 피드
- 결제, 주문, 쇼핑몰 연동
